Authentication-> Authentication is the process of verifying the identity of a user or system.
It ensures that only authorized users can access certain resources or perform specific actions. Common authentication methods include username and password, multi-factor authentication (MFA), biometric verification, and token-based authentication. 
Implementing strong authentication mechanisms is crucial for maintaining security and protecting sensitive information in web applications.

Authorization-> Authorization is the process of determining what actions or resources a user is allowed to access after they have been authenticated.
It involves defining permissions and roles that dictate what users can do within an application. Common authorization methods include role-based access control (RBAC), attribute-based access control (ABAC), and policy-based access control (PBAC).
Proper authorization ensures that users can only perform actions that they are permitted to, enhancing security and preventing unauthorized access to sensitive data or functionality.

Hashing- Hashing is a one-way cryptographic function that transforms input data (like a password) into a fixed-size string of characters, which is typically a hash value.
 This process is irreversible, meaning you cannot retrieve the original input from the hash. Common hashing algorithms include SHA-256, bcrypt, and Argon2.

salting- password salting is a technique used to enhance the security of stored passwords by adding a unique random value, known as a "salt," to each password before hashing it.
This ensures that even if two users have the same password, their hashed values will be different due to the unique salts. Salting helps protect against rainbow table attacks and makes it significantly harder for attackers to crack passwords.

passport.initialize() is a middleware that initializes Passport and is required for using Passport in an Express application. It sets up the necessary configurations and prepares Passport to handle authentication requests.
passport.session() is a middleware that integrates Passport with the session management of an Express application. It
allows Passport to maintain user authentication state across multiple requests by storing user information in the session. This enables features like persistent login sessions, where users remain authenticated even after navigating to different pages or refreshing the browser.
